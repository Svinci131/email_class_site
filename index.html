Class Outline

Over View (est 5-10min)

  • Take a Poll of Class
    (see if people are more interested in learning HTML & CSS, or just how to build emails)
  • Quickly show some beautiful emails

HTML and Layouts (est 30min)

  • Basic Tags: H1, H2, Ul, Li, ect.
  • Web Layouts: divs, sections, articles
  • Email Layouts: tables
  • Briefly touch on tags that don't work on email

Exercise: Build out the layout for our demo using using a table or divs, depending on focus of the class

Images (est 10min)

  • File size increases the likelihood of your email being sent to spam
  • Show how to write an image tag
  • With emails all image tag src attributes must lead to an online image, not to your images on your computer

Exercise: Add an image as a banner to our page

*Note never put critical info(text) in an image

Styling part I:Inline Sytle/Basic CSS (est 15 min)

  • Explain briefly what styles are
  • Change the font colors using inline styles

Exercise: Have them finish styling the rest of the email

Styling part II:Classes (est 15 min)

  • Go over Classes
  • Go over the difference between styling for the web vs styling for emails
    • Inline styles vs classes
    • Do Not use anything beyond CSS2 in email
    • Point them Web V. Email Best Practices Resource

Testing & Sending/Working with Different Email Clients: (est 20 min)

  • Use a free email previewer(probably: sendinblue) to preview email we've been building
  • Go over the ways different email clients behave
  • Give this example:
    • When we preview our email we'll see it looks strange in Outlook.
    • This is because we used margin 0 auto to center everything. Outlook doesn't recognize margins.
    • We could fix this by removing the margin and adding padding left, but then it won't always be centered on mobile devices.
    • We could add classes in and use media queries to change our padding depending on the width of the screen. However, sometimes browsers strip out classes.
    • The important thing is to know you're audience, test your emails, and prioritize what's most important.

Exercise: Upload a list(class emails) and send out our email

Web V. Email: Cheat Sheet

Website:

  • Always use classes to do your styling
  • Write your CSS in an external page and link to it from your HTML
  • Save out the images your using in the same folder as your html doc, the you got them from may take them down
  • Use divs and sections to break up your page
  • Use margins and padding and to position things,
  • Always include , html tags, a head section and a body section; (this website has folders you can download with the correct file structure for web all set up.)

Email:

  • Do all your styles inline
  • Use classes very sparingly
  • If you have to use Classes, include them in a style tag at the head of your email (YOU CANNOT LINK TO ANYTHING OUTSIDE YOUR HTML FILE)
  • Only link to images online, for the same reason, you can only use the code from your HTML file
  • Use tables within tables to position or you can use padding- donâ€™t add margins to the content with in your tables
  • You don't need to include a header, some email clients will strip it out

* A good rule of thumb is be ok with the way your email looks sans-classes
